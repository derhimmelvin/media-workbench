## 1. 环境检查

运行：

```bash
bash scripts/check-env.sh
```

需要全部满足：

- Python 3.10 或更高版本。
- Node.js 20.19 或更高版本。
- FFmpeg 可执行文件。
- 后端依赖安装后，`yt-dlp`、`FastAPI`、`keyring` 可导入。

如果本机默认 `python3` 低于 3.10，但已安装 Python 3.11，可指定解释器运行：

```bash
PYTHON_BIN=python3.11 bash scripts/check-env.sh
```

当前后端开发环境使用 `backend/.venv/bin/python`，验收时版本为 Python 3.11。

## 2. 后端

```bash
python3.11 -m venv backend/.venv
backend/.venv/bin/python -m pip install -U pip
backend/.venv/bin/python -m pip install -r backend/requirements-dev.txt
backend/.venv/bin/python -m uvicorn app.main:app --reload --app-dir backend --host 127.0.0.1 --port 8000
```

后端数据默认写入：

- `backend/data/app.sqlite3`
- `downloads/`

可通过环境变量覆盖：

- `BILI_DOWNLOADER_DATA_DIR`
- `BILI_DOWNLOADER_DATABASE`
- `BILI_DOWNLOADER_DOWNLOAD_DIR`

## 3. 前端

```bash
cd frontend
npm install
npm run dev
```

Vite 会将 `/api` 和 `/ws` 代理到 `127.0.0.1:8000`。

## 4. 验收路径

更细的场景矩阵和回归项见 [测试验收清单.md](测试验收清单.md)；面向非技术用户的操作解释见 [用户使用指南.md](用户使用指南.md)。

1. 打开 `http://127.0.0.1:5173`。
2. 同意合规声明。
3. 检查顶部状态条，确认 SQLite、yt-dlp、FFmpeg、钥匙串状态。
4. 可选：打开 Cookie 抽屉，保存 B站 Cookie；公开视频下载不强制要求 Cookie。
5. 输入公开视频链接或 BV号并解析。
6. 在资源选择区按需勾选视频、音频、封面：
   - 下载视频时需要同时选择音频流，保持合并开启可生成有声音的视频文件。
   - 仅下载音频时可选择 `m4a` 或 `mp3`。
   - 封面为可选项，可单独下载。
7. 选择视频容器格式；当前支持 `mp4` 和 `mkv`。
8. 提交下载任务，观察任务队列进度、状态和下载时间戳。
9. 下载完成后可点击任务右侧的打开目录按钮，检查保存路径中的输出文件。

## 5. 当前 v1 边界

已实现核心闭环：合规确认、Cookie 手动导入、单链接解析、视频/音频/封面选择、音视频合并、音频 MP3 转换、SQLite 任务队列、WebSocket 进度、任务清空、完成任务打开目录和基础设置。

当前暂未实现：批量导入、字幕下载/转换/翻译、封面嵌入视频元数据、打包、扫码登录、多语言界面、Redis/Celery、真实多任务并发执行。

阶段一已完成验收：

- 手工验收：公开视频下载已通过。
- 自动化浏览器响应式验收：桌面和移动宽度下无文本溢出已通过。

暂缓验收项：

- 会员/Cookie 授权资源场景。
- 不同网络下 412/403 风控场景。
