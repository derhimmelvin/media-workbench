# 测试验收清单

本文用于阶段一回归测试、手工验收和后续阶段变更检查。

## 1. 自动化测试命令

后端：

```bash
backend/.venv/bin/python -m pytest backend/tests
```

前端：

```bash
cd frontend
npm test
npm run build
```

## 2. 环境检查

```bash
bash scripts/check-env.sh
```

应确认：

- Python 版本满足 3.10+。
- Node.js 版本满足 20.19+。
- FFmpeg 可执行文件存在。
- 后端依赖可导入。
- keyring 后端可用。

如果默认 `python3` 低于 3.10：

```bash
PYTHON_BIN=python3.11 bash scripts/check-env.sh
```

## 3. 手工验收主路径

1. 运行 `bash scripts/dev.sh`。
2. 打开 `http://127.0.0.1:5173`。
3. 确认合规弹窗持续停留，不会自动消失。
4. 点击“同意并使用”。
5. 确认顶部状态条能展示 SQLite、yt-dlp、FFmpeg、钥匙串、合规确认、Cookie 状态。
6. 输入公开视频链接并解析。
7. 确认封面、标题、UP主、时长、视频流、音频流展示正常。
8. 勾选视频和音频，保持合并开启，下载为 `mp4`。
9. 下载完成后确认视频有画面且有声音。
10. 点击打开目录按钮，确认能打开输出目录。
11. 点击清空记录，确认已完成任务从任务队列消失，本地文件仍保留。

## 4. 资源选择矩阵

| 场景 | 预期 |
|---|---|
| 仅勾选视频 | 前端或后端阻止提交，提示需要同时选择音频 |
| 勾选视频 + 音频，合并开启 | 输出有声音的视频文件 |
| 勾选视频 + 音频，合并关闭 | 分别输出视频流和音频流 |
| 仅勾选音频，输出 `m4a` | 只下载音频，不下载视频 |
| 仅勾选音频，输出 `mp3` | 只下载音频，并通过 FFmpeg 转为 MP3 |
| 仅勾选封面 | 只下载封面图片 |
| 勾选音频 + 封面 | 输出音频和封面，不下载视频 |
| 勾选视频 + 音频 + 封面 | 输出视频相关文件和封面 |
| 三类资源都不勾选 | 禁止提交，提示至少选择一个资源 |

## 5. 容器与格式

| 项目 | 当前支持 | 不支持 |
|---|---|---|
| 视频容器 | `mp4`、`mkv` | `flv` |
| 音频输出 | `m4a`、`mp3` | 其他音频格式 |
| 封面 | 原始图片下载 | 封面嵌入视频元数据 |

## 6. 任务队列验收

应覆盖：

- 新任务显示下载时间。
- 完成任务显示完成时间。
- 运行中任务可取消。
- 失败或取消任务可重试。
- 完成任务可打开目录。
- 清空记录只删除已完成、失败、取消任务。
- 清空记录不删除本地下载文件。
- WebSocket 进度能更新前端状态。
- 快速刷新页面后，任务列表仍能从 SQLite 恢复。

## 7. Cookie 验收

基础验收：

- 未配置 Cookie 时，Cookie 状态显示未配置。
- 保存 Cookie 后，状态显示已配置和脱敏摘要。
- 删除 Cookie 后，状态回到未配置。
- SQLite 不保存 Cookie 明文。

暂缓验收：

- 会员/Cookie 授权资源下载。
- 不同网络下的 412/403 风控场景。

## 8. 响应式验收

已完成验收项：

- 桌面宽度下核心页面无明显文本溢出。
- 移动宽度下核心页面无明显文本溢出。

后续回归建议检查：

- 链接输入框长 URL。
- 资源选择卡片中的长格式名称。
- 任务标题较长时的展示。
- Cookie 抽屉中的长 Cookie 文本。
- 设置区域的下载路径文本。

## 9. 后端单元测试覆盖方向

当前应持续覆盖：

- URL 支持范围。
- 格式列表归一化。
- 文件名清洗。
- Cookie 脱敏。
- FFmpeg 缺失提示。
- 任务状态流转。
- 清空已结束任务。
- 拒绝 video-only 下载任务。
- 容器格式不允许 `flv`。
- 封面下载参数处理。

## 10. 阶段一结论

阶段一已完成：

- 公开视频手工下载验收通过。
- 浏览器响应式验收通过。
- 自动化测试命令可用于后续回归。

阶段一暂缓：

- 会员/Cookie 授权资源。
- 不同网络下的 412/403。
