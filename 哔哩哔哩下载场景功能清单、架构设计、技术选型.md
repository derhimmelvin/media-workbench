# 当前 v2.1 功能清单

该文档以当前阶段二 2.1 实现为准，同时保留后续扩展方向。当前产品是本地 Web 版 B站下载器：FastAPI 后端、Vue 3/Vite 前端、SQLite 本地任务队列、yt-dlp 解析下载、FFmpeg 音视频合并。

阶段二 2.1 已完成：

- 阶段一核心闭环：单链接解析、资源选择、下载、合并、任务进度和基础设置。
- 批量文本解析：多行输入、50 条上限、原始输入去重、规范化 URL 去重和逐行资源设置。
- 自定义文件名：支持单次/批量逐行提交时指定文件名；未指定时使用解析标题，并处理非法字符和文件名冲突。
- 合规增强：未同意合规声明前，解析和提交下载任务都会被后端拒绝。

后续暂缓验收：

- 会员/Cookie 授权资源场景。
- 不同网络下 412/403 风控场景。

## 一、资源解析与预览

1. 链接解析

    - 支持输入B站视频链接、BV号、av号、ep号、ss号，自动识别并提取视频元数据。

    - 解析后呈现视频标题、封面、时长、UP主等基础信息。

2. 可用资源列表展示

    - 当前 2.1 清晰列出可下载的三类资源：视频流、音频流、封面。

    - 视频/音频资源标注格式、编码、文件大小（估算）、清晰度等可用信息；封面作为可选资源单独勾选。

3. 清晰度与编码选择（视频/音频）

    - 视频：列出 yt-dlp 解析到的可用画质和编码信息。

    - 音频：列出 yt-dlp 解析到的可用音频流，并支持 `m4a` 原始输出或转为 `mp3`。

    - 支持选择具体视频流和音频流。

    - 提供 Cookie 手动导入口；扫码登录暂未实现。

## 二、下载资源选择与处理

1. 自定义勾选下载

    - 用户可自由勾选需要下载的资源：视频、音频、封面。

    - 支持部分选择，例如仅下载音频和封面。

    - 下载视频时必须同时选择音频流，避免生成“有画面无声音”的视频文件。

2. 视频与音频合并（关键）

    - 当勾选视频时，必须同时选择音频流，并自动调用合并引擎（FFmpeg）生成带声音的视频文件。

    - 支持自定义合并后的容器格式：MP4、MKV。

3. 音频输出

    - 仅下载音频时，可选择 `m4a` 或 `mp3`。

    - `mp3` 输出依赖 FFmpeg 转码。

4. 封面下载

    - 封面为可选资源，可单独下载，也可与视频/音频一起下载。

    - 当前 2.1 暂未实现封面嵌入合并后视频文件的元数据/缩略图。

---

## 三、输出与文件管理

1. 输出目录与命名

    - 支持在设置中配置下载保存路径。

    - 输出文件名支持用户自定义，并过滤非法文件名字符。

    - 完成任务支持点击按钮直接打开下载文件所在目录。

    - 如果目标文件名已存在，会自动追加序号，避免覆盖已有文件。

2. 压缩打包

    - 当前 2.1 暂未实现 ZIP、7z、tar\.gz 打包。

    - 当前 2.1 暂未实现单资源打包。

3. 自定义文件名

    - 当前 2.1 已支持单次/批量逐行提交时填写自定义文件名。

    - 用户未指定时，默认使用解析后的标题命名。

---

## 四、任务队列管理

1. 单链接任务输入

    - 当前 2.1 支持单链接解析与提交下载任务。

    - 当前 2.1 支持批量文本输入、自动去重和逐行资源设置。

    - 批量功能复用现有普通任务，不新增批量任务表。

2. 下载队列与状态控制

    - 下载任务持久化到 SQLite。

    - 当前执行器为本地后台 worker 池，按同时下载任务数设置执行任务级并发下载。

    - 支持取消、失败重试、完成任务打开目录。

    - 支持清空已完成、失败、取消的任务记录；不会清除正在执行的任务或本地下载文件。

    - 实时显示任务状态、进度、速度、剩余时间、错误信息和下载时间戳。

3. 断点续传与任务持久化

    - 所有下载任务状态实时保存至 SQLite。

    - 底层下载由 yt-dlp 执行，已启用续传参数；具体断点续传能力依赖 yt-dlp 与资源端支持。

4. 缓存与增量处理

    - 当前 2.1 暂未实现独立缓存索引与增量复用。

    - 缓存策略配置暂未实现。

## 五、用户体验与效率功能

1. 当前工作台

    - 首屏即本地工具工作台，不做营销页。

## 六、系统与合规保障

1. 启动合规声明

    - 首次进入页面强制展示免责弹窗：“本工具仅供学习、研究和个人合理使用，严禁用于侵犯版权或商业传播，用户须自行承担法律责任。” 需用户同意后方可使用。

    - 弹窗不会自动消失；用户不点击“同意并使用”时，弹窗持续停留。

2. 凭证与隐私安全

    - 用户登录 Cookie 通过系统钥匙串保存。

    - SQLite 仅保存 Cookie 是否配置和脱敏摘要，不保存明文 Cookie。

    - 如果系统钥匙串不可用，后端拒绝明文落盘并返回可见错误。

    - 翻译 API Key 当前 2.1 暂未涉及。

3. 请求频率与反爬限制

    - 解析与下载基于 yt-dlp，并使用浏览器化请求头降低 412 Precondition Failed 的概率。

    - 会员/Cookie 授权资源场景、不同网络下 412/403 风控场景暂缓验收。

4. 设置中心

    - 当前可配置项：下载保存路径、默认视频格式、默认音频格式、同时下载任务数。

    - 缓存目录、默认打包格式、翻译引擎及密钥、网络代理等暂未实现。

# 架构设计和技术选型

## 一、总体架构设计

当前 2.1 采用本地 Web 分层架构：Vue 3/Vite 前端通过 HTTP API 与 WebSocket 连接 FastAPI 后端；后端封装 yt-dlp、FFmpeg、SQLite、系统钥匙串和本地后台任务执行器。后续仍可演进为更完整的微核心架构。

架构全景图（文字描述）：

```Plain Text
┌─────────────────────────────────────────────────┐
│                  前端界面 (UI)                    │
│  Web / Electron / CLI 均可适配                    │
└─────────────────┬───────────────────────────────┘
                  │  HTTP / WebSocket
┌─────────────────▼───────────────────────────────┐
│               API 网关与服务层                    │
│  (FastAPI)                                      │
│  - 任务提交、查询、控制                           │
│  - 资源预览、配置管理                             │
│  - WebSocket 推送实时进度                        │
└─────────────────┬───────────────────────────────┘
                  │
┌─────────────────▼───────────────────────────────┐
│              任务调度与消息队列                    │
│  (v1: 本地后台线程 + SQLite 队列)                  │
│  - 按同时下载任务数执行下载任务                   │
│  - 支持取消、重试、清空已结束任务                  │
│  - Celery / Redis 作为后续扩展                    │
└─────────────────┬───────────────────────────────┘
                  │
┌─────────────────▼───────────────────────────────┐
│              核心处理引擎 (Engine)                 │
│  ┌───────────────────────────────────────────┐  │
│  │  解析器 (Extractors)                       │  │
│  │  - YtDlpBilibiliExtractor                 │  │
│  │  - 预留 YouTube / TikTok 接口              │  │
│  └───────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────┐  │
│  │  下载器 (Downloader)                       │  │
│  │  - yt-dlp 下载与续传参数                    │  │
│  │  - 速度限制、代理支持作为后续规划             │  │
│  └───────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────┐  │
│  │  媒体处理器 (Media Processor)              │  │
│  │  - 视频/音频合并 (FFmpeg)                  │  │
│  │  - 音频 MP3 转换 (FFmpeg)                  │  │
│  │  - 字幕处理作为后续规划                      │  │
│  └───────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────┐  │
│  │  打包器 (Packer)                           │  │
│  │  - v1 暂未实现，后续支持 ZIP / 7z / tar.gz  │  │
│  └───────────────────────────────────────────┘  │
└─────────────────────────────────────────────────┘
                  │
┌─────────────────▼───────────────────────────────┐
│              数据与存储层                         │
│  - SQLite : 设置、合规确认、任务、任务事件        │
│  - 系统钥匙串 : Cookie 明文仅保存在这里             │
│  - 文件系统 : 下载目录                            │
```

关键解耦原则：

当前后端通过 `BaseExtractor` 抽象隔离平台解析与下载实现；API 层负责 Web 交互；任务执行器负责本地异步执行耗时下载操作。

## 二、技术栈详细建议

|层级|推荐技术|理由|
|---|---|---|
|前端|Vue 3 \+ Vite \+ TypeScript \+ Element Plus \+ Pinia|当前已实现，适合本地功能密集型工具面板|
|后端 API|Python 3\.11 \+ FastAPI|当前已实现，提供 HTTP API、OpenAPI 文档和 WebSocket|
|任务队列|本地后台 worker 池 \+ SQLite 队列|当前 2.1 已实现任务级并发、状态持久化、取消、重试、清空已结束任务|
|核心引擎|yt-dlp \+ `YtDlpBilibiliExtractor`|当前已实现，复用 yt-dlp 的 Bilibili extractor，同时保留 `BaseExtractor` 抽象|
|媒体处理|FFmpeg 命令行|当前用于音视频合并和 MP3 转换|
|凭证存储|keyring / macOS 系统钥匙串|当前用于保存 B站 Cookie；SQLite 只保存脱敏摘要|
|数据库|SQLite 本地单文件|当前用于保存设置、合规确认、任务、任务事件|
|部署|本地开发脚本|当前通过 `scripts/dev.sh` 启动前后端；Docker、Redis、Celery 暂未引入|
|后续规划|Redis/Celery、字幕转换、翻译服务、打包库、代理配置|这些能力不属于当前 2.1|

## 三、核心流程与数据设计

核心流程图：

```Plain Text
用户提交链接
    ↓
解析链接 (Extractor)
    ↓
展示资源列表 → 用户勾选视频、音频、封面并选择格式
    ↓
提交下载任务 → 本地后台执行器接手
    ↓
┌── 按同时下载任务数执行任务，按任务参数下载视频、音频、封面
│   ↓
├── 若视频+音频 → 调用 FFmpeg 合并
│   ↓
├── 若仅音频且选择 mp3 → 调用 FFmpeg 转码
│   ↓
├── 记录进度、速度、状态、错误与时间戳
│   ↓
└── 任务完成，通知前端，更新历史记录
```

数据库关键表设计（SQL）：

- `settings`：用户配置键值对，如下载路径、默认视频格式、默认音频格式、同时下载任务数、Cookie 脱敏摘要。

- `compliance_consent`：合规声明确认状态、版本和确认时间。

- `tasks`：任务ID、链接、标题、状态、进度、阶段、输出目录、输出路径、选流参数、创建时间、更新时间、完成时间、错误信息。

- `task_events`：任务状态事件，用于记录任务阶段、进度和消息。

缓存策略：

- 当前 2.1 未实现独立缓存索引。

- 后续可按 `(video_id, stream_id)` 建立缓存键，避免重复下载和重复后处理。

## 四、关键设计决策与注意事项

### 视频合并与性能

- 合并视频依赖 FFmpeg，由 yt-dlp 下载视频流和音频流后完成封装。

- 当前视频容器支持 MP4、MKV；FLV 已移除。

- 仅下载音频时可保留 m4a，也可通过 FFmpeg 转为 mp3。

### 断点续传实现

- 当前 2.1 使用 yt-dlp 的续传能力，不自行实现 HTTP `Range` 细节。

- 对于 B站视频流，其 URL 可能带时效性签名；失败时可通过重试重新触发解析与下载。

### 安全与隐私

- B站 Cookie 使用系统钥匙串保存。

- SQLite 只保存 Cookie 配置状态和脱敏摘要，不保存明文 Cookie。

- 合规弹窗：进入页面时强制显示，并记录同意状态，不允许跳过直接使用。

### 多平台扩展设计

- 定义 `BaseExtractor` 抽象类，要求实现：

    - `supports(url) -> bool`

    - `fetch_info(url, auth) -> dict`

    - `download(task, auth, progress_hook) -> str`

- 当前实现为 `YtDlpBilibiliExtractor`，内部封装 yt-dlp 的 Bilibili extractor。

- 未来增加 YouTube 支持时，可新增对应 Extractor，并在路由层增加注册与选择逻辑。

### 前端交互优化点

- 进度反馈：通过 WebSocket 实时推送任务速度、百分比，以及当前阶段（等待、下载中、处理中、完成、失败、取消）。

- 资源选择可视化：通过视频、音频、封面各自的下载勾选框表达选择状态；已移除重复的底部选择状态条。

- 任务操作按钮保留悬浮提示，悬浮框显示在按钮下方，避免快速滑动残影。

- 模板管理暂未实现。

## 五、部署与运维建议

当前 2.1 本地启动：

- 后端：`backend/.venv/bin/python -m uvicorn app.main:app --reload --app-dir backend --host 127.0.0.1 --port 8000`

- 前端：`cd frontend && npm run dev`

- 统一启动：`bash scripts/dev.sh`

后续如进入服务化部署，可再引入 Docker Compose、Redis、Celery Worker 和反向代理。

开发顺序建议：

1. 已完成本地 Web 核心闭环：解析、选择、下载、合并、任务进度、设置。

2. 已完成 FastAPI API、SQLite 队列、本地后台执行器、WebSocket 推送。

3. 已完成 Vue 3/Vite 工作台，集成合规、Cookie、解析、资源选择、任务队列。

4. 后续逐步补充字幕下载/转换、翻译、打包、缓存和单任务内部资源并发。

5. 后续增强会员/Cookie 授权资源验收、不同网络下 412/403 场景处理。

6. 后续扩展预设模式、多语言界面、桌面端封装等体验功能。

---

以上架构和技术选型能够支撑当前阶段二 2.1 实用增强，并保留未来成长为多平台下载工具的可能性。
